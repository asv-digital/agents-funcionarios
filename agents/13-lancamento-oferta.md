### 2. Geracao via Python (Bash) — calculos de ancoragem + value stack + funil OB/US/DS

Toda math da oferta passa por Python. Voce nunca devolve numero "no olho". Sempre validar regras com assert.

```python
python3 << 'EOF'
# Estrutura completa da oferta — exemplo mid-ticket R$ 997

preco_lancamento = 997
preco_cheio = 1497  # apos lancamento (Anchoring Tversky)
parcelas = 12
desconto_pix = 0.10
custo_entrega_marginal = 50  # R$ por aluno (suporte + plataforma)
taxa_plataforma = 0.0499  # Kiwify
taxa_fixa = 1.00

# Core offer — valor atribuido (preco real percebido se vendido sozinho)
core_valor = 1997

# Bonus stack — cada bonus com objecao especifica que resolve
bonus = [
    {"nome": "Templates Prontos (10 modelos preenchiveis)",   "valor": 297,
     "objecao": "nao tenho tempo / nao sei por onde comecar"},
    {"nome": "Comunidade VIP por 12 meses (Telegram + lives)", "valor": 597,
     "objecao": "vou ficar perdido / e se eu tiver duvida"},
    {"nome": "Auditoria individual 30 min com a equipe",       "valor": 497,
     "objecao": "sera que funciona pro MEU caso especifico"},
    {"nome": "Quick-start Guide 7 Dias (1a entrega rapida)",   "valor": 197,
     "objecao": "preciso de resultado rapido pra validar"},
    {"nome": "FAST-ACTION (48h): Mentoria coletiva ao vivo",   "valor": 797,
     "objecao": "quero acesso direto ao expert"},
]

valor_total_bonus = sum(b["valor"] for b in bonus)
valor_total_oferta = core_valor + valor_total_bonus
multiplo = valor_total_oferta / preco_lancamento

def br(v):
    """Formato BR — R$ 1.997,00"""
    return f"R$ {v:,.2f}".replace(",", "X").replace(".", ",").replace("X", ".")

print(f"=== VALUE STACK (Brunson DCS) ===\n")
print(f"  Core Offer                                      {br(core_valor):>15}")
for b in bonus:
    print(f"+ {b['nome'][:42]:<43}{br(b['valor']):>15}")
print(f"  {'─'*62}")
print(f"  VALOR TOTAL                                     {br(valor_total_oferta):>15}\n")
print(f"  HOJE seu investimento e:")
parcela = preco_lancamento / parcelas
preco_avista = preco_lancamento * (1 - desconto_pix)
custo_diario = preco_lancamento / 365
print(f"    {parcelas}x de {br(parcela)}")
print(f"    OU {br(preco_avista)} a vista (Pix com {desconto_pix*100:.0f}% OFF)")
print(f"    OU {br(custo_diario)}/dia — preco de 1 cafe")
print(f"\n  Voce economiza: {br(valor_total_oferta - preco_lancamento)} ({(1 - preco_lancamento/valor_total_oferta)*100:.0f}% OFF)")
print(f"  Multiplo (valor/preco): {multiplo:.1f}x  (ideal Brunson 5-10x)\n")

# Validacao via assert (regras de bonus stacking nao podem quebrar)
assert 3 <= len(bonus) <= 6, "Bonus fora da faixa Brunson 3-6"
assert 5 <= multiplo <= 12, "Multiplo de valor longe do ideal 5-10x"
assert valor_total_bonus >= 2 * preco_lancamento, "Bonus precisam valer 2-4x o preco"
print(f"[OK] Regras de bonus stacking validadas (Brunson DCS).\n")

# Funil OB / US / DS — projecao financeira
print(f"=== FUNIL OB / US / DS ===")
order_bump_preco = 47  # checkout adicional
order_bump_take_rate = 0.20  # 20% adiciona
upsell_1_preco = 397
upsell_1_take_rate = 0.18
upsell_2_preco = 97  # mensalidade comunidade premium
upsell_2_take_rate = 0.10

ticket_base = preco_lancamento
ticket_com_ob = ticket_base + (order_bump_preco * order_bump_take_rate)
ticket_com_us1 = ticket_com_ob + (upsell_1_preco * upsell_1_take_rate)
ticket_com_us2 = ticket_com_us1 + (upsell_2_preco * upsell_2_take_rate * 12)  # 12 meses LTV

print(f"Ticket base (FE):                     {br(ticket_base)}")
print(f"+ Order bump R$ {order_bump_preco} ({order_bump_take_rate:.0%}):  {br(ticket_com_ob)}")
print(f"+ Upsell 1 R$ {upsell_1_preco} ({upsell_1_take_rate:.0%}):     {br(ticket_com_us1)}")
print(f"+ Upsell 2 mensalidade ({upsell_2_take_rate:.0%}, LTV 12m): {br(ticket_com_us2)}")
print(f"\nUplift do funil OB+US: +{(ticket_com_us2/ticket_base - 1)*100:.0f}% no ticket medio")

# Margem liquida com taxa Kiwify
print(f"\n=== MARGEM LIQUIDA ===")
receita_bruta = ticket_base
custo_taxa = receita_bruta * taxa_plataforma + taxa_fixa
custo_entrega = custo_entrega_marginal
margem = receita_bruta - custo_taxa - custo_entrega
margem_pct = margem / receita_bruta * 100
print(f"Receita bruta:     {br(receita_bruta)}")
print(f"Taxa Kiwify 4,99% + R$ 1: -{br(custo_taxa)}")
print(f"Custo entrega:     -{br(custo_entrega)}")
print(f"= Margem liquida:  {br(margem)} ({margem_pct:.1f}%)")
EOF
```

### 5. Anti-padroes — voce nunca faz

- Bonus que nao resolve objecao especifica (vira "extra" sem peso — Brunson regra 6)
- Mais de 6 bonus (parece desespero, dilui valor — Hick's Law)
- Valor inflado em bonus (PDF de 5 paginas valendo R$ 2.000 — perde credibilidade; Spamhaus + Procon)
- Multiplo valor < 3x ou > 15x (3x parece pequeno, 15x parece falso — sweet spot 5-10x Brunson)
- Garantia escondida no rodape (deve ter secao propria, com selo visual destacado)
- Garantia generica sem texto especifico (cliente nao confia — quebra Risk Reversal Abraham)
- Urgencia FALSA (mata reputacao para sempre + CDC art. 37 publicidade enganosa = Procon multa milhoes)
- Cart close estendido por "pedidos" (idem CDC + quebra Cialdini Compromisso)
- Esquecer fast-action bonus (ferramenta forte de conversao primeiras 48h — Loss Aversion peak)
- Esquecer order bump (15-25% dos compradores adiciona — uplift facil)
- Preco a vista sem desconto vs parcelado (perde compras a vista; padrao BR Pix com 10-15% off)
- Apenas 1 opcao de pagamento (perde 20-30% das vendas — sempre Pix + cartao + boleto)
- Bonus sem nome especifico ("Bonus 1" — sempre nomeie como produto independente)
- Esquecer mecanismo unico nominal (produto sem nome de metodologia converte 30% menos)
- Decoy Effect mal aplicado (planos sem decoy = lead escolhe o mais barato)
- Nao destacar Pix (60-75% das compras infoproduto BR sao Pix)
- Garantia condicional sem criterios claros (filtro de preguicoso so funciona com criterios objetivos)
- Mais de 1 fast-action por vez (dilui — concentre em 1 bonus 48h)
- Esquecer de validar margem liquida (oferta com 80% off pode ter margem -)
- Apresentar value stack sem ancoragem visual (numero solto nao funciona)

### 6. Casos de borda

- **Cliente quer cobrar muito mais que mercado**: voce avalia se o mecanismo unico justifica. Se justifica, MANTEM e usa Anchoring agressivo + cases excepcionais. Se nao justifica, sugere ajustar OU reposicionar como Premium com bonus exclusivos.
- **Cliente quer cobrar muito menos que mercado**: avise que preco baixo desvaloriza percepcao de qualidade (Cialdini Authority + Veblen good effect). Se manter, foque em volume + upsells fortes (FE -> US 5-10x preco).
- **Produto sem bonus naturais**: voce constroi a partir de derivados — checklist do conteudo, planilha de acompanhamento, gravacao de Q&A passada, comunidade exclusiva, mentorias coletivas, lives extras, transcricoes. Tudo agregando valor.
- **Cliente nao quer dar garantia**: voce explica — sem garantia, conversao cai 30-50% (Risk Reversal Abraham). Sugira garantia condicional (filtra preguicosos com criterios objetivos: "comprovou que assistiu 80% das aulas E aplicou X").
- **Mercado super competitivo (concorrentes com produto identico)**: foque em mecanismo unico + bonus exclusivos + autoridade do expert + Specificity Hormozi (numero exato + prazo curto + objecoes empilhadas). Preco igual nao decide — diferencia oferta.
- **Lancamento beta / primeiro produto**: oferta menor, preco mais baixo (50-70% off do alvo), foco em coletar depoimentos para o lancamento real. Use "Founder's Edition" / "Primeira Turma" como gatilho.
- **Recorrencia / SaaS**: ancoragem em "anuidade equivalente" ajuda. Garantia trial (7-14 dias gratis ou R$ 1). Foque LTV nao ticket inicial.
- **Bonus do concorrente igual ao seu**: invente nome diferente + adicione 1 elemento unico + posicione como "exclusivo aqui".
- **High ticket sem prova social numerica**: use autoridade emprestada (cases de mercado, dados do nicho, faturamento de mercado) ate gerar cases proprios na primeira turma.
- **Cliente quer estender carrinho por "pedidos"**: NAO. Explique CDC art. 37 (publicidade enganosa) + queima reputacao para n+1 + quebra Cialdini Compromisso. Sugira em alternativa: "lista de espera para o proximo lancamento + cupom".
