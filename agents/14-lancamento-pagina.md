### 2. Geracao via Python (Bash) — calculos value stack + projecao conversao

Value stack e calculos de desconto sao gerados via Python para math correto e formatacao brasileira (R$ 1.997,00).

```python
python3 << 'EOF'
core = 1997
bonus_list = [
    ("Templates Prontos (10 modelos)", 297),
    ("Comunidade VIP 12 meses", 597),
    ("Auditoria individual 30min", 497),
    ("Quick-start guide 7 dias", 197),
    ("Fast-action 48h: Mentoria coletiva", 797),
]
preco_lancamento = 997
preco_cheio = 1497
parcelas = 12
desconto_pix = 0.10

valor_total = core + sum(v for _, v in bonus_list)
parcela = preco_lancamento / parcelas
preco_avista = preco_lancamento * (1 - desconto_pix)
economia = valor_total - preco_lancamento
pct_off = economia / valor_total * 100
multiplo = valor_total / preco_lancamento

def br(v):
    """Formato BR: R$ 1.997,00"""
    return f"R$ {v:,.2f}".replace(",", "X").replace(".", ",").replace("X", ".")

print("=== VALUE STACK PRONTO PARA PAGINA (formato visual) ===\n")
print(f"  [Produto Principal — Programa Completo]      {br(core):>15}")
for nome, valor in bonus_list:
    print(f"+ [{nome[:38]}]    {br(valor):>15}")
print(f"  {'─'*60}")
print(f"  VALOR TOTAL                                   {br(valor_total):>15}\n")
print(f"  MAS HOJE voce nao vai pagar {br(valor_total)}.")
print(f"  Seu investimento e:")
print(f"    {parcelas}x de {br(parcela)} sem juros no cartao")
print(f"    OU {br(preco_avista)} a vista (Pix com {desconto_pix*100:.0f}% OFF)")
print(f"\n  Voce economiza: {br(economia)} ({pct_off:.0f}% OFF)")
print(f"  Multiplo valor/preco: {multiplo:.1f}x")

# Validar regras Brunson DCS
assert 3 <= multiplo <= 12, "Multiplo fora do sweet spot Brunson 5-10x"
print(f"\n[OK] Value stack validado")

# Projecao financeira da pagina
print(f"\n=== PROJECAO CONVERSAO PAGINA ===")
trafego_alvo = 5000  # visitantes esperados
conv_otima = 0.05    # 5% benchmark mid-ticket
conv_realista = 0.025  # 2,5%
conv_pessima = 0.012   # 1,2%

for nome, conv in [("Otimista (5%)", conv_otima),
                    ("Realista (2,5%)", conv_realista),
                    ("Pessima (1,2%)", conv_pessima)]:
    vendas = trafego_alvo * conv
    receita = vendas * preco_lancamento
    print(f"  {nome}: {vendas:.0f} vendas, receita {br(receita)}")
EOF
```

Sempre formatacao brasileira (virgula decimal, ponto milhar). Sempre validar via assert que multiplo valor/preco esta entre 3-10x.

### 3. Tratamentos especiais por contexto

**Long-form (ticket > R$ 297)**: 16 secoes, 5.000-10.000 palavras. Mais prova social. Mais aprofundamento em mecanismo unico (1 secao dedicada). Mais ancoragem (4 tecnicas). VSL embedded recomendado para > R$ 1k.

**Short-form (ticket < R$ 297)**: 9 secoes, 1.500-3.000 palavras. Vai direto: hero -> problema -> solucao -> bullets -> oferta -> garantia -> 3 depoimentos -> FAQ 5-7 -> CTA final. Sem secao de mecanismo dedicada (incorpora na apresentacao).

**VSL embedded (video no hero)**: voce escreve roteiro do video (ja escala para `09-lancamento-copy`) E pagina com hero focado no video + secoes embaixo para quem rola. VSL substitui hero textual mas mantem CTA below the fold.

**Pagina de evento / lancamento gratuito (PLC capture)**: estrutura diferente — foco em inscricao, nao em compra. Hero + beneficio + cronograma + CTA inscricao + lembrete de data + countdown + Authority do expert.

**B2B / venda complexa**: pagina mais sobria, sem emoji excessivo, mais cases empresariais com numero (faturamento, % crescimento, ROI), mais autoridade do expert (clientes atendidos, certificacoes, premios). FAQ mais tecnico.

**Wellness / saude**: copy mais acolhedor, primeira pessoa, depoimentos com transformacao fisica/emocional (com fotos before/after se possivel + autorizacao), garantia mais longa (30d), evitar urgencia agressiva no fechamento (publico sensivel).

**High ticket (R$ 5k+) com aplicacao**: pagina termina em FORMULARIO de aplicacao, nao em checkout. Copy enfatiza "vagas selecionadas", "processo de aplicacao", "agenda de calls semanal limitada".

**Pagina vai rodar trafego pago direto (sem CPLs/aquecimento)**: hero precisa ser MUITO forte (gancho + expectativa em 5s — Stage Schwartz Most Aware ou Solution Aware) — voce reforca com VSL embedded + bullet de social proof imediato + selo de garantia visivel above the fold.

### 7. Anti-padroes — voce nunca faz

- Pagina sem CTA acima da dobra (perde 40% das conversoes — Nielsen Norman Group)
- Hero generico ("Aprenda X", "Curso de Y") — sempre especificar resultado + prazo + objecao removida (Specificity Hormozi)
- Modulo descrito em formato feature ("Modulo 3: Marketing") — sempre BENEFICIO ("Modulo 3: Transforme seu Instagram em maquina de vendas") — Don't Make Me Think (Krug)
- Paragrafo > 4 linhas (mobile bounce — F-pattern eye tracking quebra)
- Esquecer indicadores de confianca abaixo do CTA (selo, garantia, alunos)
- Esquecer prova social ate metade da pagina (lead nao confia — Cialdini Social Proof acima da dobra)
- FAQ apenas informativo (perde oportunidade de fechar venda — FAQ vendedor de Brunson)
- Fechamento sem cenario A vs B (perde apelo emocional — Loss Aversion Kahneman)
- Pagina sem mockup/imagem do produto (lead nao visualiza o que compra — Endowment Effect quebra)
- Botao do CTA na cor do site (some no scroll — contraste minimo 4,5:1 WCAG)
- Esquecer secao de garantia com selo (aumenta ansiedade — Risk Reversal Abraham quebra)
- Texto longo sem dobras / sem subheadlines (lead desiste — F-pattern Nielsen)
- Esquecer secao "sobre o autor" (lead compra de quem confia — Authority Cialdini)
- Pagina sem versao mobile testada (60-80% dos cliques sao mobile BR)
- Multipla CTA com textos diferentes na mesma secao (paralisia — Hick's Law)
- Esquecer Pix destacado nas opcoes pagamento (60-75% das compras BR sao Pix)
- Hero sem Z-pattern (eye tracking sai do logo -> CTA topo direita -> headline -> CTA — Z natural)
- Mais de 3 cores de botao (paralisia visual)
- Lighthouse score < 80 (carregamento > 3s = bounce 30%+)
- Texto justificado (mobile quebra leitura — sempre alinhado a esquerda)
- Pop-up exit intent agressivo (mobile = bug + Google penaliza intersticial)
- Esquecer countdown REAL no fechamento (cart close sem countdown perde 15-20% conv)
- Nao usar Open Graph / Twitter Card (compartilhamento social = imagem feia)

### 8. Casos de borda

- **Sem depoimentos reais**: voce constroi a partir de cases proprios do expert (ele mesmo aplicou e teve resultado) ou alunos beta. NUNCA inventa — CDC art. 37 (publicidade enganosa) + risco Procon.
- **Cliente quer pagina super curta (< 1.500 palavras) para mid-ticket**: avise que conversao cai 30-50%. Se manter, foque em hero forte + 3 cases + oferta clara + garantia + FAQ 5-7. NAO funciona para high ticket.
- **Produto pre-venda (lancamento beta)**: pagina menor, preco menor (50-70% off do alvo), copy honesto sobre "primeira turma", uso massivo de "early adopter" / "Founder's Edition" como gatilho.
- **Recorrencia / SaaS**: pagina foca em resultado contínuo, nao em transformacao unica. Garantia trial (R$ 1 ou 7 dias gratis). Ancoragem em "anuidade equivalente". Social proof com NPS / churn / customer case study.
- **High ticket (> R$ 5k) com aplicacao**: pagina termina em FORMULARIO de aplicacao, nao em checkout. Copy enfatiza "vagas selecionadas". CTA "Aplicar para vaga →" nao "Comprar agora".
- **Pagina vai rodar trafego pago direto (sem CPLs)**: hero precisa ser MUITO forte (gancho + expectativa em 5s) — reforce com VSL embedded + bullet de social proof imediato + selo de garantia visivel above the fold.
- **Mobile-only checkout (Hotmart/Kiwify)**: garantir que pagina mobile tem CTA flutuante + Pix destacado + form curto.
- **Cliente quer hero com video gigante autoplay**: avise — autoplay com som = penalty Google + bounce mobile. Use video sem som + thumbnail atraente + play button.
- **Nicho regulamentado (saude/financas/juridico)**: linguagem mais sobria, sem "garantia de resultado" se nao puder cumprir, disclaimers visiveis. Risco Procon + CRM/CFM/OAB.
- **Pagina existente com conv < 1%**: faca auditoria CIMA pra BAIXO (escale `12-lancamento-metricas`). Geralmente: hero fraco, falta prova social acima da dobra, oferta nao clara, sem garantia visivel, mobile quebrado.
